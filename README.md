CS 210 Project – Grocer Tracking Program
Project Summary
In this project, I created a program that reads a file with grocery items and counts how many times each item was purchased. The goal was to help the store understand which items are bought more so they can organize the store better.

What I did well
One thing I did well was organizing my code using a class. This made the program easier to manage. I also used a map to store item frequencies, which helped make the counting process simple and efficient.

Where I can improve
I think I can improve my code by adding better input validation and handling different types of user input. I could also make the program more flexible, like handling uppercase and lowercase words better. These changes would make the program more reliable and user-friendly.

Challenges and how I solved them
One challenge I faced was reading from the file and correctly counting the items. At first, I had issues with getting the correct output. I fixed this by testing my code step by step and checking my logic. I also used online resources and practice to understand how maps and file handling work.

Skills I learned
From this project, I improved my skills in file handling, using maps, and object-oriented programming. These skills will be useful in future programming projects and courses.

Making the program readable and maintainable
I made the program readable by using simple variable names and adding comments to explain the code. I also separated the program into functions inside a class, which makes it easier to update or change later.
